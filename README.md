# Cifragmenter
Инструмент для поиска молекул и лигандов в cif фалах

CIFragmenter — алгоритм для автоматизированной фрагментации CIF-файлов с сохранением геометрии органических молекул и координационных лигандов в формате mol. 

## Установка
1. Установите python3.12
2. Скачайте папку проекта
3. В терминале перейдите в папку проекта
4. Запустите установку

```python
./cifragmenter setup
```

## Использование
1. В папку проекта добавьте папку, содержащую cif файлы
2. Запустите фрагментацию командой

```python
./cifragmenter run <input_dir> --fragment-type <coord/mols>
```
- <input_dir> - имя папки с cif файлами 
- <coord/mols> - режим фрагментации: поиск «уникальных молекул» (mols) или «лигандов с разрывом координационных связей» (coord)

### Режим поиска «уникальных молекул»
В этом режиме будут найдены все уникальные молекулы, входящие в кристаллические структуры, и сохранены в папке done в формате mol и sdf.

### Режим поиска  «лигандов с разрывом координационных связей»
В этом режиме будут найдены все лиганды, координированные атомами металлов. Информация о природе и объеме полиэдра Вороного-Дирихле металлических атомов, с которыми были связаны донорные атомы, а также о длине и значениях телесных углов связей будет сохранена в свойствах файло mol и sdf.

### Дополнительные настройки фрагментации
- --ccdc-chemical-name: метка строки cif файла, содержащей название вещества (по умолчанию _chemical_name_systematic)
- --db-code-pattern: метка строки cif файла, содержащей Ref-код структуры в базе данных (по умолчанию _database_code_depnum_ccdc_archive)
- --min-occ: пороговое значение степени заселенности атомных позиций. Позиции, содержащие элементы с заселенностью ниже этого порога, исключаются из дальнейшего анализа, так как, вероятно, соответствуют частично занятым позициям (по умолчанию 0,5)
- --property: метка строки cif файла, содержащей информацию о каком-либо свойстве вещества (по умолчанию meelting_point)
- --timeout: максимальное время в секундах, затрачиваемое на фрагментацию одного cif файла. Файлы, время обработки которых превышает эту величину будут пропущены и перемещены в папку timeout

## Сохранение результата фрагментации
В папке с cif файлами будут созданы:
- папка "done": в нее будут перемещены успешно обработанные файлы
- папка "error": в нее будут перемещены файлы, обработка которых не удалась
- папка "timeout": в нее будут перемещены файлы, на обработку которых ушло времени больше, чем указано в --timeout
- папка "mols": в ней будут сохранена геометрия найденных молекул или лигандов в формате mol
- файл "SDF.sdf": набор молекул и свойств
